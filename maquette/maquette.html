CE QUE CHAQUE ZONE FAIT
Frontmatter YAML — la matière première de la machine : Dataview et le cahier-maître lisent statut/tags ; le resume: est LA ligne que Claude récolte pour bâtir l'index.
LaTeX inline — rendu nativement par Obsidian, mélangé au Markdown dans le même fichier. Acquis V1 pour le carnet technique.
Embed ![[…]] — l'image vit dans captures/, rangée et légendée par Claude au moment du drop (pas d'OCR : on range, on ne lit pas).
Tâche datée — case à cocher Markdown standard ; le plugin Tasks la centralise dans le cahier-maître, Claude peut la lister/trier par échéance.
Wikilink [[…]] — backlink automatique côté cible : la couche « réseau » façon NotebookLM, posée par l'humain ou par Claude.
La couche machine — _index.ai.md — ce que Claude lit à la place du contenu complet :

- Cuisine/2026-06-10-plan-cuisine.md — calcul poutre porteuse + dimensions [cuisine,structure] (en-cours)

Trois lignes au lieu de trois pages → économie de tokens.

Une page = un fichier Markdown lisible sans aucun logiciel. C'est ce qui rend le double accès possible : l'humain et Claude travaillent sur exactement la même matière.
